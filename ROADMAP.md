# Roadmap

The roadmap is milestone based; dates are intentionally avoided until maintainer
capacity and legal ownership are settled.

## 1. Foundation

- Adopt charter, licenses, schema, contribution and security policies.
- Establish four protected GitHub repositories and asset ownership records.
- Deploy a static preview with the complete moderation path disabled by default.

## 2. Validation

- Manually research at least 50 diverse records.
- Test corrections, expiry, disputes, provider submissions, and appeals.
- Define freshness targets per opportunity category.

## 3. Private pilot

- Invite a small contributor and moderator group.
- Operate Supabase intake with retention limits and audit logging.
- Measure review accuracy, processing time, and contributor friction.

## 4. Public launch

- Publish the static index, open dataset, policies, and transparency pages.
- Complete accessibility, performance, security, and recovery reviews.
- Publish sponsorship disclosures before accepting any sponsorship.

## 5. Stewardship

- Formalize the appropriate legal entity before material recurring funds.
- Add independent reviewers and a documented appeals process.
- Reassess architecture only from observed scale or reliability needs.
