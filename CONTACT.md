# Contact PerkCommons

Use the address that best matches the request so it reaches the appropriate
maintainer without unnecessary access to private information.

| Address | Purpose |
| --- | --- |
| [`hello@perkcommons.com`](mailto:hello@perkcommons.com) | General questions and project contact |
| [`support@perkcommons.com`](mailto:support@perkcommons.com) | Help using the site, submissions, and reports |
| [`security@perkcommons.com`](mailto:security@perkcommons.com) | Private vulnerability and security reports |
| [`privacy@perkcommons.com`](mailto:privacy@perkcommons.com) | Privacy, deletion, and GDPR-related requests |
| [`press@perkcommons.com`](mailto:press@perkcommons.com) | Media and press enquiries |
| `admin@perkcommons.com` | Internal administration; not a public support channel |
| `noreply@perkcommons.com` | Reserved for future automated mail; replies are not monitored |

Never send passwords, authentication tokens, raw IP addresses, or unnecessary
personal data by email. Vulnerabilities should follow the private process in
[`SECURITY.md`](SECURITY.md). Privacy requests are handled under
[`PRIVACY.md`](PRIVACY.md).
